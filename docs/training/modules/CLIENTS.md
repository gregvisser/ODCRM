# Clients

## Purpose
Use `OpensDoors Clients` to confirm the base client record you are working on before you enter Marketing or Onboarding. This is the safest place to answer: "Am I working on the right client?"

## You are here in the workflow
Use this tab:
- before onboarding a new client,
- before changing marketing setup for a client,
- when reports, inbox, or lead data look like they belong to the wrong client.

After this tab, users normally go to:
- `Onboarding` for setup work,
- `Readiness` for live operational status,
- `OpensDoors Marketing` for actual outreach work.

## When a user should use this tab
Use this tab when you need to:
- confirm you are working on the correct client,
- review or update account details,
- review contacts,
- inspect lead-side context before moving into marketing work.

## Before you start
- You are signed in.
- You know which client you are supposed to work on.

## What the user sees on screen
Current sub-tabs:
- `Accounts`
- `Contacts`
- `Leads`

## What each sub-tab is for
| Sub-tab | What it is for | What it is not for |
|---|---|---|
| `Accounts` | Confirming the client record and core account details. | Launching or monitoring outreach. |
| `Contacts` | Reviewing saved contacts for the selected client. | Working live inbox replies. |
| `Leads` | Reviewing client-side lead-related information. | Replacing the `Lead Sources` setup flow. |

## Recommended operator path
1. Open `OpensDoors Clients`.
2. Open `Accounts`.
3. Confirm the client name and key details.
4. Open `Contacts` if you need to check the people attached to that client.
5. Open `Leads` only if you need client-side lead context before moving into marketing work.
6. Move to `Onboarding` or `OpensDoors Marketing`.

## What to click and what to expect
### Confirm the correct client
1. Open `Accounts`.
2. Review the visible client/account details.
3. Make sure the client you see is the client you intended to work on.

Expected result:
- you know the client context is correct before you enter the tenant-scoped marketing tabs.

### Check contacts before outreach work
1. Open `Contacts`.
2. Review whether the expected contacts exist.
3. Note any missing or outdated contact information before continuing.

Expected result:
- you know whether the contact data is ready for later onboarding or marketing work.

## What happens after each action
- Reviewing data here does not start sending.
- This tab reduces downstream mistakes by helping you confirm client scope before you work in marketing.

## How this tab connects to other tabs
- `Clients` -> `Onboarding`: for detailed setup.
- `Clients` -> `Email Accounts`, `Lead Sources`, `Suppression List`, `Templates`, and `Sequences`: only after the client context is confirmed.
- `Clients` -> `Readiness`: when you want to check live operational state after confirming client context.

## Common mistakes / failure states / confusion points
- Treating `Clients` as the place to launch or monitor outreach.
- Assuming the `Leads` sub-tab replaces the `Lead Sources` setup process in Marketing.
- Forgetting to confirm the correct client before entering marketing tabs.

## How to verify success
You are done with this tab when:
- you know which client you are working on,
- the account and contact context looks correct enough to proceed,
- you know whether the next step is setup work or operational work.

## What to do next
- Go to [Onboarding](./ONBOARDING.md) if setup is incomplete.
- Go to [Readiness](./READINESS.md) if the client is already operating.
- Go to [Email Accounts](./EMAIL_ACCOUNTS.md) or [Lead Sources](./LEAD_SOURCES.md) if marketing prerequisites still need work.

## Reality check notes
- The current training set is centered on operator workflows in Marketing and Onboarding. `Clients` is still important because it anchors those workflows to the correct client record.
