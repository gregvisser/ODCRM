# Clients

## Purpose
Use `OpensDoors Clients` to confirm the base client record you are working on, review contacts, and inspect lead-related information before you move into Marketing or Onboarding.

## When a user should use this tab
Use this tab when you need to:
- confirm you are working on the correct client,
- review or update account details,
- review contacts,
- inspect the client-side data before moving into Marketing.

## Prerequisites
- You must be signed in.
- You should know which client you intend to work on.

## What the user sees on screen
Current sub-tabs:
- `Accounts`
- `Contacts`
- `Leads`

Use this area as the data-health workspace, not the sending workspace.

## Main actions available
### Accounts
- select and review a client/account,
- inspect account details,
- navigate into related areas such as Marketing or Onboarding from account context.

### Contacts
- review saved contacts for the current client.

### Leads
- review lead-related information for the current client.

## Step-by-step common workflow
1. Click `OpensDoors Clients`.
2. Open `Accounts`.
3. Confirm the client name and core details are correct.
4. Open `Contacts` and make sure the right people exist.
5. Open `Leads` only if you need to inspect lead-side context before going to Marketing.
6. Move to `Onboarding` for setup work or `Marketing` for outreach work.

## What happens after each action
- Reviewing data here does not start sending.
- Navigation from this area helps you confirm client scope before working in tenant-scoped Marketing tabs.

## How this tab connects to other tabs
- `Clients` -> `Onboarding`: for detailed setup.
- `Clients` -> `Marketing`: once the base client data is correct.
- `Clients` -> `Readiness`: when you want a quick operational status after confirming the client.

## Common mistakes / failure states / confusion points
- Treating `Clients` as the place to launch sending. It is not.
- Assuming that a lead-related view here replaces the Marketing `Lead Sources` flow. It does not.
- Forgetting to confirm the correct client before entering Marketing.

## Operational tips
- Use this tab first when you are not sure whether a data problem is really a Marketing problem.
- Treat it as the client record checkpoint before doing anything higher risk.

## Reality check notes
- The current training set is centered on the operator workflow in Marketing and Onboarding. `Clients` is still important because it anchors that workflow to the right client record.
